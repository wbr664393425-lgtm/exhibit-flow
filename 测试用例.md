> 版本：v1.0

## 说明：状态码速查

| 实体 | 状态值 | 含义 |
|------|--------|------|
| `eh_apply.status` | 0 | 草稿 |
| `eh_apply.status` | 1 | 待审批（审批中） |
| `eh_apply.status` | 2 | 已通过 |
| `eh_apply.status` | 3 | 已驳回 |
| `eh_apply.status` | 4 | 已取消 |
| `eh_apply.status` | 5 | 已改期 |
| `eh_approval_node.status` | pending | 当前待审批 |
| `eh_approval_node.status` | waiting | 等待中（前置节点未通过） |
| `eh_approval_node.status` | approved | 已通过 |
| `eh_approval_node.status` | rejected | 已驳回 |
| `eh_approval_node.status` | transferred | 已转交 |
| `eh_visit_record.status` | 0 | 待录入 |
| `eh_visit_record.status` | 1 | 已录入 |
| `eh_visit_record.status` | 2 | 已归还 |

| 属性 | 值 |
|------|----|
| 类型 | 正常用例 |
| 接口 | `POST /eh/apply/draft` |
| 权限 | `eh_apply_add` |

**请求体**

```json
{
  "title": "TC001-测试草稿申请",
  "startDate": "2026-05-10",
  "startHour": "9",
  "endHour": "11",
  "unit": "阿里云科技有限公司",
  "industry": "互联网",
  "headCount": 8,
  "leader": "副总裁",
  "applicant": "张三",
  "dept": "销售部",
  "phone": "13800138001",
  "district": "西湖区",
  "agenda": "产品对接与方案演示",
  "services": ["桌牌打印", "茶水"],
  "visitors": [
    {
      "name": "李四", "title": "总经理",
      "unit": "阿里云科技有限公司",
      "isStrategic": false, "strategicLevel": ""
    }
  ]
}
```

**预期结果**

- 响应：`{"code":0,"data":true}`
- DB 断言：
  ```sql
  SELECT status FROM eh_apply WHERE subject='TC001-测试草稿申请' ORDER BY id DESC LIMIT 1;
  -- 期望: status = '0'
  ```
- DB 断言：`eh_approval_node` 中该 apply_id **无记录**（草稿不创建审批节点）
- DB 断言：`eh_apply_visitor` 中该 apply_id **有 1 条**来访客户记录

### TC-010 时段冲突检测（有冲突）

| 属性 | 值 |
|------|----|
| 类型 | 正常用例 |
| 接口 | `GET /eh/apply/conflict` |
| 前置 | 存在 status=1 的申请，start_time=2026-05-10 09:00，end_time=2026-05-10 11:00 |

**请求参数**：`date=2026-05-10&startHour=10&endHour=12`（时段重叠：10:00-12:00 与 09:00-11:00 重叠）

**冲突判定逻辑**：`reqStart(10:00) < existEnd(11:00)` AND `reqEnd(12:00) > existStart(09:00)` → 冲突

**预期结果**

- 响应：`{"code":0,"data":"TC002-直接提交申请（09:00 - 11:00）"}`（返回冲突申请描述字符串）

**请求体**

```json
{
  "applyId": "<TC002的申请ID>",

}
```

**预期结果**

- 响应：`{"code":0,"data":true}`
- DB 断言（节点1）：`status='approved', action='agree', opinion='同意，请展厅主管审批'`
- DB 断言（节点2）：`status='pending'`（一级通过后二级激活）
- DB 断言（申请）：`status='1'`（还有待处理节点，申请仍为审批中）
- DB 断言（通知）：`eh_notification` 新增一条 `title='申请进入下一审批环节'`

## 模块八：完整业务闭环（端到端）

---

### TC-E2E-001 展厅申请全流程闭环

> 覆盖：申请提交 → 二级审批全通过 → 参观留存录入 → 照片上传 → 归还签收

| 步骤 | 操作 | 验证点 |
|------|------|--------|
| 1 | `POST /eh/apply/submit`（TC-002 body） | `eh_apply.status='1'`，2 个审批节点已创建 |
| 2 | 一级审批 approve（TC-101） | 节点1 approved，节点2 pending，`eh_apply.status='1'` |
| 3 | 二级审批 approve（TC-102） | 节点2 approved，`eh_apply.status='2'`，通知已发 |
| 4 | 录入参观留存（TC-201） | `eh_visit_record.status='1'`，actual_count 同步到 apply |
| 5 | 上传现场照片（TC-301） | `eh_visit_photo` 有 2 条新记录 |
| 6 | 归还签收（TC-203） | `eh_visit_record.status='2'`，apply 的 return_signer 已更新 |

**DB 最终状态断言**

```sql
-- 申请状态
SELECT status, actual_count, return_signer FROM eh_apply WHERE id = <apply_id>;
-- 期望: status='2', actual_count=7, return_signer='李主管'

-- 审批节点全通过
SELECT node_level, status FROM eh_approval_node WHERE apply_id = <apply_id>;
-- 期望: 2条记录均 status='approved'

-- 留存状态
SELECT status, opportunity_code FROM eh_visit_record WHERE apply_id = <apply_id>;
-- 期望: status='2', opportunity_code='OPP-2026-001'

-- 照片数量
SELECT COUNT(*) FROM eh_visit_photo WHERE visit_id = <visit_id>;
-- 期望: 2
```
