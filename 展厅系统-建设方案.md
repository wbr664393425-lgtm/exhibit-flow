展厅访问申请与留存系统 建设方案

一、背景与目标

当前展厅使用流程依赖线下 docx 申请表 + 线上 Excel
参观数据表,两端割裂,流程缺失闭环。

申请端:展厅使用申请.docx ——
填写会议主题/使用时间/来访单位/陪同领导等,线下走审批。

留存端:展厅参观数据表.xlsx ——
事后登记参观客户、战客级别、商机编码、现场照片等,定期报送省公司战客管理系统。

**主要痛点:**

1.  申请与留存两张表割裂,数据无法联动,客户信息重复录入。

2.  审批无流转节点、无提醒、无冲突检测(同一时段展厅可能被重复预约)。

3.  参观后商机编码、现场照片补录无载体,Excel 用 DISPIMG
    嵌入图片难以管理。

4.  取消/改期/超时归还等异常情况无闭环处理。

5.  报送省公司时需人工整理 Excel,全年战客覆盖率(≥50%)等指标无自动统计。

**目标:建设一套 申请 → 审批 → 参观 → 留存 → 商机跟进 → 报送
全链路线上系统,替代双表离线流转。**

二、角色与权限

| **角色** | **核心动作** | **使用端** |
|----|----|----|
| 申请人(区县/部门同事,多区县共用一个账号) | 提交申请、查看进度、取消/改期、录入参观结果、补传照片与商机 | H5 为主 |
| 部门领导 | 一级审批(申请部门领导) | PC |
| 展厅主管 | 二级审批、排期冲突确认、归还签收 | PC |
| 分管副总 / 总经理 | 三级审批(按最高陪同领导字段动态触发) | PC |
| 展厅管理员 Admin | 排期总览、异常处理、基础数据维护、数据报送导出 | PC |
| 省公司查看员 | 只读统计看板、覆盖率指标 | PC |

三、技术方案

脚手架与框架

- 脚手架:Pig4Cloud(Spring Cloud + Vue3)

- 后端:Spring Boot + MyBatis-Plus + Pig RBAC + 状态字段驱动审批,遵循阿里
  Java 规范

- PC 端(管理/审批/看板/报表):Vue3 + Element Plus + FullCalendar

- 移动 H5 端(区县申请人):Vue3 + Vant,独立部署,共用 Pig
  后端接口,微信/手机浏览器访问

- H5
  端页面:共享账号登录、新建申请、我的申请、申请详情(进度/取消/改期)、参观结果录入、现场照片拍摄上传、商机补录

- 存储:MySQL 8、MinIO(照片)、Redis

- 通知:仅站内消息(Pig 消息中心 / WebSocket)

- 对外数据:Apache POI + EasyExcel 导出,列结构严格对齐
  展厅参观数据表.xlsx Sheet2

建表规范

所有业务表必含四字段:create_by VARCHAR(64)、update_by
VARCHAR(64)、create_time DATETIME、update_time DATETIME,逻辑删除
del_flag。

关键表

| **表名** | **说明** |
|----|----|
| eh_apply | 申请主表:会议主题/议程/起止时段/陪同领导级别/申请人账号/真实姓名/手机号/所属区县/申请部门/状态/附加服务 |
| eh_apply_visitor | 参观客户明细(1 对多,对齐多客户多行) |
| eh_approval_node | 审批节点流水(动态审批链) |
| eh_visit_record | 参观留存(按客户拆分,字段对齐 xlsx Sheet2) |
| eh_visit_photo | 现场照片(MinIO 路径) |
| eh_opportunity | 商机跟进(允许后期补录) |
| eh_change_log | 改期/取消/变更流水 |
| sys_dict | 省代码/地市/行业/战客级别(Pig 字典模块) |

docx 存档

通过 poi-tl 按原模板回填生成审批通过后的 docx 存档版。

四、分期交付

| **阶段**     | **周期** | **内容**                                            |
|--------------|----------|-----------------------------------------------------|
| 阶段 1 (MVP) | 2–3 周   | 申请 + 审批 + 排期冲突 + 参观留存(含照片)+ 基础字典 |
| 阶段 2       | 2 周     | 取消/改期/超期异常闭环、商机补录、站内消息完善      |
| 阶段 3       | 2 周     | 统计看板、覆盖率指标、省公司报送导出、docx 回填     |
| 阶段 4       | 按需     | BI 深化、移动端适配优化                             |

五、已确认决策

- 技术栈:Pig4Cloud(PC 管理端)+ Vue3/Vant 独立 H5 端(区县申请端)

- 覆盖范围:单一展厅

- 账号策略:多区县共用一个申请人账号,表单内用 所属区县 + 真实姓名 +
  手机号 区分实际填报人

- 省公司对接:仅 Excel 报送(模板严格对齐 xlsx Sheet2)

- 通知渠道:仅站内消息(不接入企微/短信)

六、验证方式

6.  按 docx 字段提交申请,审批链跑通 3 级,产出审批通过记录。

7.  同展厅同时段二次申请被拦截。

8.  取消/改期生成变更流水且排期释放。

9.  留存录入后导出 Excel,列结构与 展厅参观数据表.xlsx Sheet2
    完全一致,可直接导入战客管理系统。

10. 全年战客覆盖率计算与人工抽样核对一致。
